Alguns pontos importantes sobre o elemento br:

  • Ele representa uma quebra de linha;
  • É um elemento do tipo void (não possui tag de fechamento);
  • Quebra o texto na linha que foi inserido;
  • Útil para organizar o texto em linhas dentro de parágrafos ou outros elementos.

Elemento hr

Parágrafo de texto 2

O elemento hr significa "horizontal rule", representa uma régua horizontal.

Algumas características do elemento hr:

  • Insere uma linha horizontal para dividir conteúdos;
  • Elemento do tipo void;
  • Ocupa toda a largua disponível por padrão;
  • Aceita alguns atributos para pesonalização:
    • size - Define a altura da linha em pixels;
    • width - Define a largura da linha em pixels ou em porcentagem;
    • color - Define a cor da linha.

Formulários

Estrutura básica de um formulário

É composta por:

  • Tag FORM: delimita o formulário com os atributos ACTION e METHOD
  • Campos para entrada de dados: inputs de diferentes tipos
  • Botão de envio: input do tipo submit

Tag FORM

Usado para criar um formulário. Contém todos os demais campos e botões dentro dele.

O formulário precisa definir para onde os dados serão enviados, isso é feito pelo atributo action. Geralmente se passa um URL para um script que irá processar as informações no servidor.

Também é necessário definir o method, que determina se os dados serão enviados pelo método GET ou POST do protocolo HTTP.

Exemplo de tag FORM

Campos com INPUT

Para inserir os campos onde o usuário poderá digitar ou selecionar informações utilizamos a tag input que possui vários tipos diferentes, como text, checkbox, radio, submit, etc.

Exemplo

Nome: Email:

O atributo name serve para identificar cada campo.

Botão de Envio

O botão input type="subimit" é responsável por finalizar o envio do formulário.

Sem ele, o usuário não tem como disparar o envio das informações preenchidas para o servidor.

Atributos da tag FORM

  • target - define em qual janela o resultado será exibido, podendo ser _self, _blank, _parent, _top;
  • enctype - especifica o tipo de codificação dos dados enviados. Mais usado em forms com envio de arquivos;
  • autocomplete - indica se o navegador deve preencher os campos automaticamente com base em dados anteriores;
  • novalidate - desabilita a validação HTML5 dos campos ao enviar o formulario.

Métodos HTTP

GET

O método GET deve ser usado em formulários que apenas "obtém" dados do servidor, sem alterações.

Exemplos: buscas, filtros ou qualquer outro formulário que apenas recupera informações.

POST

O POST é utilizado quando o formulário resulta em uma atualização de dados no servidor. Como um cadastro, envio de mensagens ou modificações no banco de dados.

Uma diferença importante é que os dados de um GET ficam expostos na URL, enquanto no POST não. O POST deve ser usado para envio de informações sensíveis, como senhas.

Campos de formulário com INPUT

  • text - campo de texto simples, com uma linha
  • password - igual ao text mas oculta a digitação com asteriscos
  • email - campo otimizado para e-mails
  • number - aceita apenas números
  • checkbox - caixas de seleção para escolha múltipla
  • radio - botões de opção para escolha única
  • submit - botão de envio do formulário
  • file - campo para envio de arquivos

Atributos comuns da tag input:

  • name - nome do campo
  • value - valor padrão
  • placeholder - texto de exemplo dentro do campo
  • required - torna o preenchimento obrigatório
  • disabled - desabilita o campo
  • checked - marca por padrão (checkbox e radio)
  • accept - tipos de arquivos permitidos (file)
  • maxlength - tamanho máximo do texto

Recursos Modernos do HTML5

Elementos Semanticos

Cabeçalho <header>

O elemento header é usado para definir o cabeçalho de uma página ou seção. Normalmente contém logotipos, títulos e menus de navegação.

Navegação <nav>

O elemento nav define um bloco de navegação com links de uma página ou seção. Normalmente usado para menus de sites e aplicações web.

  • Sobre
  • Produtos
  • Contato

Conteúdo Principal <main>

O elemento main serve para englobar o conteúdo principal de uma página, aquele relacionado diretamente ao tópico central. Cada página deve ter apenas um main.

Artigos <article>

O elemento article representa conteúdo independente que faz sentido por si só, como posts de blog, notícias e cards de produtos.

Seções <section>

O elemento section é usado para agrupar conteúdos relacionados ao mesmo assunto em uma página, como capítulos de um livro.

Rodapé <footer>

O elemento footer contém informações de rodapé de uma página ou seção, como informações do autor, links úteis, direitos autorais, etc.
