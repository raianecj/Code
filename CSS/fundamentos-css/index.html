Entendendo as cores no CSS

Trabalhando com códigos RGB

Significa Red, Green e Blue. O formato do código RGB é:

rgb(red, green, blue). Onde:

  • red: é o valor da cor vermelha, de 0 a 255
  • green: é o valor da cor verde, de 0 a 255
  • blue: é o valor da cor azul, de 0 a 255

Trabalhando com códigos Hexadecimais

Funciona com uma base 16, utilizando os números de 0 a 9 e as letras de A a F para representar os valores de 0 a 15. O formato do código é:

#RRGGBB. Onde:

  • RR - Dois digitos Hexadecimais para o vermelho
  • GG - Dois digitos Hexadecimais para o verde
  • BB - Dois digitos Hexadecimais para o azul

Exemlos

  • #FF0000 - Vermelho
  • #00FF00 - Verde
  • #0000FF - Azul

Trabalhando com Códigos HSL

O HSL significa "Hue, Saturation e Lightness", que traduzindo do inglês representam Matiz, Saturação e Luminosidade.

Permite selecionar uma cor a partis de:

  • Matiz: é o tom puro da cor, variando de 0 a 360 graus em um círculo cromático. 0 representa o vermelho, 120 representa o verde e 240 representa o azul.
  • Saturação: representa a "pureza" ou intensidade da cor, variando de 0% a 100%. Quanto menor a saturação, mais "lavada" fica a cor.
  • Luminosidade: representa a quantidade de luz, variando de 0% (preto) a 100% (branco). 50% representa a cor pura sem escurecer ou clarear.

O formato do códgo é:

hsl(hue, saturation, lightness). Onde:

  • hue - Matiz, de 0 a 360
  • saturation - Saturação, de 0% a 100%
  • lightness - Luminosidade, de 0% a 100%

Estilizando Textos e Fontes com CSS

Propriedades de Texto

Vamos começar vendo algumas das principais propriedades de texto:

text-align

Serve para alinhar horizontalmente o texto dentro de um elemento. Os valores mais comuns são:

  • left: Alinha o texto à esquerda.
  • right: Alinha o texto à direita.
  • center: Centraliza o texto.
  • justify: Alinha o texto de forma justificada, distribuindo igualmente ao longo de toda a largura.

text-decoration

Adiciona decorações visuais no texto, como linhas, sublinhados etc. Os valores mais utilizados são:

  • none: Sem decoração (remove sublinhados de links).
  • underline: Sublinha o texto.
  • line-through: Um risco no meio do texto.
  • overline: Uma linha acima do texto.

text-transform

Transforma o texto entre maiúsculas e minúsculas. Os valores disponíveis são:

  • uppercase: Todo o texto ficará maiúsculo.
  • lowercase: Todo o texto ficará minúsculo.
  • capitalize: Primeiras letras maiúsculas.

Propriedades de Fonte

font-family

Define a fonte a ser utilizada no texto. Você pode especificar o nome da fonte ou uma categoria genérica. Alguns nomes genéricos de fontes:

  • serif - fontes com serifa, com pequenos traços nas extremidades.
  • sans-serif - fontes sem serifa.
  • monospace - fonte de largura fixa, ideal para códigos.
  • cursive - imita uma caligrafia.
  • fantasy - fontes decorativas, geralmente usadas em títulos.

p { font-family: Arial, Helvetica, sans-serif;}

font-size

Altera o tamanho do texto, podendo utilizar diversas unidades, como pixels, EM, ou porcentagem.

h1 {font-size:32px;}

font-weight

Define o peso(grossura) da fonte, aceitando palavras-chaves ou números de 100 a 900. Deixa os paragrafos em negrito. Outros pesos comuns são normal(400) e ligther (300).

p {font-weight: bold;}

font-style

Altera o estilo da fonte. Os valores disponíveis são:

  • normal - texto normal
  • italic - texto em itálico
  • oblique - texto inclinado

Estilizando com Classes CSS

Uma boa prática é criar classes CSS específicas para aplicar negrito, itálico e outros estilos, ao invés de usar as tags HTML diretamento:

.bold { font-weight: bold; }.italic { font-style: italic; }

Assim você pode utilizar da seguinte forma no HTML:

<p>Texto normal <span class="bold">texto negrito<span> mais texto...<p>

Unidades de Medida

Pixel

Um pixel representa um ponto único na tela. Em monitores, 1 pixel geralmente corresponde ao menor elemento que pode ser controlado naquele dispositivo.

Por exemplo, se temos uma div com width: 10px, ela terá uma largura fixa de 10 pixels, independente do tamanho ou zoom da tela.

Alguns casos onde os pixels são úteis:

  • Definir a largura de bordas: como as bordas geralmente são finas (1 a 3 pixels), os pixels permitem um controle preciso.
  • Tamanhos de fontes pequenas: para textos auxiliares, usar tamanho em pixels (ex: 12px) permite consistência.
  • Elementos de tamanho conhecido e fixo: em alguns casos, sabemos exatamente o tamanho que um elemento deve ter, para encapsular um conteúdo específico.

REM

REM significa "Root EM" e é uma unidade relativamente recente que veio para simplificar o uso de medidas responsivas.

O REM baseia o tamanho do elemento no tamanho da fonte definida no elemento raiz (root) do documento HTML. Por exemplo:

div { font-size: 16px; /* Tamanho padrão da fonte nos navegadores */}
section { width: 3rem; // 3 * 16px = 48px}

Como a fonte padrão nos navegadores geralmente é de 16px, 1rem corresponde a 16px. Então se usarmos 3rem, teremos 3 * 16 = 48px. A vantagem do REM é que, se o usuário alterar o tamanho de fonte padrão no navegador para facilitar a leitura, todo o layout se adaptará proporcionalmente.

Então se o root font-size passar para 20px, 3rem será equivalente a 60px. Isso faz do REM uma unidade adaptável. Por isso ela é muito utilizada para definir espaçamentos, tamanhos de container, larguras de colunas etc.

Usando valores fracionados

Também podemos usar valores fracionados em REM, como 1.5rem, 2.25rem etc. Nesses casos, a fração é calculada em cima do tamanho base de 16px (ou do root font size definido).

Então 2.5rem com root de 16px = 2.5 * 16 = 40px. Isso permite ajustar os tamanhos com mais precisão.

EM

A unidade EM funciona de forma muito parecida com o REM, mas ao invés de se basear no font size raiz, o EM utiliza como referência o font size do próprio elemento. Exemplo:

section font-size: 16px;
button { padding: 1em; // 1 * 16px do font-size do section}

Novamente, se o font-size do elemento "pai" for alterado, o em se adapta automaticamente.

Isso faz do EM uma unidade útil quando queremos que o elemento filho tenha um tamanho proporcional ao elemento pai. Como em botões com padding relativo ao text size, por exemplo.

Porcentagem

A unidade porcentagem (%) em CSS é relativa ao elemento pai, similar ao EM. A diferença é que a porcentagem se baseia no width (largura) ou height (altura) do elemento pai, enquanto o EM usa o font size. Exemplo:

main width: 800px;
section width: 50%; // 50% de 800px = 400px
parent height: 100px;
el { height: 25% // 25% de 100px = 25px }

Novamente vemos que a porcentagem cria uma proporcionalidade entre elementos pai e filho: se redimensionamos o elemento pai, os elementos filhos mudarão proporcionalmente.

Viewport: VW e VH

As unidades VW e VH são relativas à viewport, que é a área visível da página no navegador.

  • VW = Viewport Width
  • VH = Viewport Height

1VW representa 1% da largura total da viewport.1VH representa 1% da altura total da viewport. Exemplo:

div { width: 50vw; // 50% da largura da viewport height: 75vh; // 75% da altura da viewport }

Essas unidades são ideais para criar elementos com largura ou altura relativa ao tamanho da janela do navegador, o que é útil especialmente para layouts responsivos. Alguns casos de uso:

  • Hero images e banners que ocupam toda a viewport em altura
  • Layouts com colunas que devem manter proporção em diferentes resoluções
  • Elementos grandes que precisam limitar seu tamanho à viewport (imagens, vídeos etc)

Resumindo

  • Pixels: controle preciso para tamanhos fixos e pequenos elementos
  • REM: tamanhos proporcionais ao root, ideal para espaçamentos e layouts responsivos
  • EM: tamanhos relativos e proporcionais ao elemento pai
  • Porcentagem: proporcional ao width/height do elemento pai
  • VW | VH: proporcional à largura e altura da viewport

Heranças no CSS

A herança no CSS permite que valores de determinadas propriedades que são aplicadas a um elemento, sejam herdadas pelos elementos filhos. Ou seja, os elementos filhos acabam assumindo algumas características de estilo definidas no elemento pai.

Por exemplo, se definirmos uma cor de fonte vermelha para o elemento body, todos os textos contidos nele automaticamente ficarão com a cor vermelha, mesmo que não tenhamos aplicado a propriedade color diretamente a eles. Isso acontece porque a propriedade color é herdada do elemento pai body para os filhos.

Propriedades herdadas no CSS

  • color - cor do texto
  • font-family - família de fonte do texto
  • font-size - tamanho da fonte do texto
  • line-height - altura da linha do texto
  • text-align - alinhamento do texto

Ou seja, de forma geral as propriedades relacionadas a fontes e textos são herdadas no CSS.

Especificidade no CSS

A especificidade CSS determina a precedência com que as regras de estilo são aplicadas num elemento HTML.

Ela define basincamente "o quão específico" é o seletor CSS utilizado em uma regra de estilização. Exemplo:

/* Menos específico */
p { color: blue; }
/* Mais específico */
.main-text { color: red;}
/* Ainda mais específico */
#unique-element{ color: yellow; }

No exemplo acima, a regra com o ID #unique-element tem maior especificidade e será aplicada caso haja conflito com as outras duas.

Especificidade e ordem de aplicação

De maneira geral, os estilos CSS são aplicados em uma ordem de cascata, de acordo com sua posição no arquivo CSS ou no documento HTML. Porém, com a especificidade, é possível "quebrar" essa ordem e fazer com que regras definidas anteriormente sejam sobrepostas por regras especificadas depois.

Alguns pontos importantes sobre a aplicação das regras:

  • Estilos inline (direto no elemento HTML) sempre ganham dos estilos nos arquivos CSS externos, independentemente da ordem.
  • Estilos nos arquivos externos são aplicados por último e podem ser sobrescritos pelos outros.
  • Cada tipo de seletor CSS tem um nível diferente de especificidade, conforme veremos a seguir.
  • Múltiplos seletores somam especificidades. Um grupo de seletores tem especificidade maior.
  • Seletores com a mesma especificidade seguem a ordem em que aparecem no código fonte.

Especificidade dos seletores CSS

Seletor universal

O seletor universal * se aplica a todos os elementos do documento, mas tem especificidade muito baixa. Qualquer outro seletor consegue sobrepor uma regra com o seletor universal.

Seletores de tipo

São os seletores que referenciam elementos HTML por seu nome, como h1, p, div etc.

Têm especificidade maior que os seletores universais, mas menor que seletores de classe e ID.

h1 { color: green; }
.main { color: red; /* Ganha do h1 */}

Seletores de classe e atributo

São qualquer seletor que utiliza um ponto . para selecionar classes ou colchetes [] para selecionar atributos.

.main { color: green;}
[title] { color: red; /* Igual especificidade */}

Esses seletores têm maior especificidade que seletores de tipo. Uma classe ou atributo vai sempre ganhar de um seletor de elemento.

ID selector

O seletor de ID (#element) tem a maior especificidade entre seletores simples. Como um ID deve ser único em uma página, ele é muito específico e ganha das outras regras.

#unique { color: green; }h1 { color: red; /* Perde para o #unique */}

Agrupando seletores

Você também pode combinar seletores para aumentar a especificidade. Cada seletor adicional na regra soma especificidade. Por exemplo:

/* Especificidade maior */
body .content .text { color: red;}
/* Perde da regra acima */
.text { color: blue; }

O primeiro grupo combina os seletores body, classe .content e classe .text, tornando-o mais específico e dando prioridade mesmo vindo antes.

Seletores Avançados do CSS

O CSS possui diversos seletores que permitem estilizar elementos HTML de forma avançada e específica. Os mais úteis são:

  • nth-child
  • nth-of-type
  • hover
  • focus
  • first-child
  • last-child
  • not
  • before
  • after

Variáveis no CSS

As variáveis CSS permitem armazenar valores que podem ser reutilizados em todo o código CSS. Isso traz muitos benefícios, como:

  • Centralizar valores que são utilizados com frequência, como cores, espaçamentos, tamanhos de fonte etc;
  • Facilitar a alteração global desses valores em todo o projeto;
  • Tornar o código mais legível e fácil de manter.

Declarando variáveis

Para declarar uma variável utilizamos a seguinte sintaxe:

--nome-da-variavel: valor;

Onde:

  • -- : É obrigatório para indicar que estamos declarando uma variável
  • nome-da-variavel: Um nome para identificar a variável. Deve ser escrito em letras minúsculas e com hífens no lugar de espaços. Não utilize caracteres especiais.
  • valor: O valor que desejamos armazenar na variável. Pode ser qualquer valor CSS válido como cores, medidas, porcentagens etc.valor: O valor que desejamos armazenar na variável. Pode ser qualquer valor CSS válido como cores, medidas, porcentagens etc.
  • ;: O ponto e vírgula no final para finalizar a declaração.

Por exemplo, para armazenar uma cor que utilizaremos frequentemente:

--cor-principal: #5533cc;

Para armazenar um tamanho de fonte padrão:

--tamanho-padrao: 16px;

Para armazenar um espaçamentos que será reutilizado:

--espacamento-pequeno: .5rem;

Onde declarar as variáveis

Geralmente declaramos as variáveis CSS no seletor :root. O :root se refere ao elemento raiz do documento HTML e é um bom lugar para centralizar as variáveis que serão utilizadas globalmente.

:root {
--cor-principal: #5533cc;
--tamanho-padrao: 16px;
--espacamento-pequeno: .5rem;
}

Declarar no :root faz com que as variáveis fiquem disponíveis globalmente para todo o CSS.

Utilizando variáveis

Para utilizar o valor armazenado em uma variável, utilizamos a função var():

elemento {
propriedade: var(--nome-da-variavel);
}

Por exemplo, para aplicar a cor principal que armazenamos:

h1 { color: var(--cor-principal); }

Para utilizar o tamanho de fonte padrão:

p { font-size: var(--tamanho-padrao);}

E o espaçamento pequeno:

button { margin: var(--espacamento-pequeno); }

A função var() irá substituir a chamada pela variável pelo valor armazenado. Facilitando reutilizar os valores e modificá-los globalmente.

Boas práticas

  • Utilize nomes descritivos e intuitivos para as variáveis;
  • Centralize as declarações de variáveis globais no :root;
  • Utilize padrões nos nomes, como prefixos indicando o tipo de valor que está armazenado;
  • Evite nomes muito genéricos como "valor", "numero", etc;
  • Evite caracteres especiais, espaços, acentuação etc. Utilize apenas letras minúsculas, números e hífens;
  • Declare as variáveis no topo dos arquivos CSS antes de começar a utilizá-las.

Sobrescrevendo variáveis

Também é possível sobrescrever o valor de uma variável em contextos mais específicos:

:root {
--cor-principal: #5533cc; }.botao --cor-principal: #333;
}

No exemplo acima, a variável --cor-principal terá o valor #5533cc globalmente. Porém no contexto dentro do seletor .botao ela será sobrescrita com o valor #333.

Isso permite reaproveitar as variáveis globais mas alterando conforme a necessidade de elementos específicos.
