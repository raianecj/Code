Entendendo as cores no CSS

Trabalhando com códigos RGB

Significa Red, Green e Blue. O formato do código RGB é:

rgb(red, green, blue). Onde:

  • red: é o valor da cor vermelha, de 0 a 255
  • green: é o valor da cor verde, de 0 a 255
  • blue: é o valor da cor azul, de 0 a 255

Trabalhando com códigos Hexadecimais

Funciona com uma base 16, utilizando os números de 0 a 9 e as letras de A a F para representar os valores de 0 a 15. O formato do código é:

#RRGGBB. Onde:

  • RR - Dois digitos Hexadecimais para o vermelho
  • GG - Dois digitos Hexadecimais para o verde
  • BB - Dois digitos Hexadecimais para o azul

Exemlos

  • #FF0000 - Vermelho
  • #00FF00 - Verde
  • #0000FF - Azul

Trabalhando com Códigos HSL

O HSL significa "Hue, Saturation e Lightness", que traduzindo do inglês representam Matiz, Saturação e Luminosidade.

Permite selecionar uma cor a partis de:

  • Matiz: é o tom puro da cor, variando de 0 a 360 graus em um círculo cromático. 0 representa o vermelho, 120 representa o verde e 240 representa o azul.
  • Saturação: representa a "pureza" ou intensidade da cor, variando de 0% a 100%. Quanto menor a saturação, mais "lavada" fica a cor.
  • Luminosidade: representa a quantidade de luz, variando de 0% (preto) a 100% (branco). 50% representa a cor pura sem escurecer ou clarear.

O formato do códgo é:

hsl(hue, saturation, lightness). Onde:

  • hue - Matiz, de 0 a 360
  • saturation - Saturação, de 0% a 100%
  • lightness - Luminosidade, de 0% a 100%

Estilizando Textos e Fontes com CSS

Propriedades de Texto

Vamos começar vendo algumas das principais propriedades de texto:

text-align

Serve para alinhar horizontalmente o texto dentro de um elemento. Os valores mais comuns são:

  • left: Alinha o texto à esquerda.
  • right: Alinha o texto à direita.
  • center: Centraliza o texto.
  • justify: Alinha o texto de forma justificada, distribuindo igualmente ao longo de toda a largura.

text-decoration

Adiciona decorações visuais no texto, como linhas, sublinhados etc. Os valores mais utilizados são:

  • none: Sem decoração (remove sublinhados de links).
  • underline: Sublinha o texto.
  • line-through: Um risco no meio do texto.
  • overline: Uma linha acima do texto.

text-transform

Transforma o texto entre maiúsculas e minúsculas. Os valores disponíveis são:

  • uppercase: Todo o texto ficará maiúsculo.
  • lowercase: Todo o texto ficará minúsculo.
  • capitalize: Primeiras letras maiúsculas.

Propriedades de Fonte

font-family

Define a fonte a ser utilizada no texto. Você pode especificar o nome da fonte ou uma categoria genérica. Alguns nomes genéricos de fontes:

  • serif - fontes com serifa, com pequenos traços nas extremidades.
  • sans-serif - fontes sem serifa.
  • monospace - fonte de largura fixa, ideal para códigos.
  • cursive - imita uma caligrafia.
  • fantasy - fontes decorativas, geralmente usadas em títulos.

p { font-family: Arial, Helvetica, sans-serif;}

font-size

Altera o tamanho do texto, podendo utilizar diversas unidades, como pixels, EM, ou porcentagem.

h1 {font-size:32px;}

font-weight

Define o peso(grossura) da fonte, aceitando palavras-chaves ou números de 100 a 900. Deixa os paragrafos em negrito. Outros pesos comuns são normal(400) e ligther (300).

p {font-weight: bold;}

font-style

Altera o estilo da fonte. Os valores disponíveis são:

  • normal - texto normal
  • italic - texto em itálico
  • oblique - texto inclinado

Estilizando com Classes CSS

Uma boa prática é criar classes CSS específicas para aplicar negrito, itálico e outros estilos, ao invés de usar as tags HTML diretamento:

.bold { font-weight: bold; }.italic { font-style: italic; }

Assim você pode utilizar da seguinte forma no HTML:

<p>Texto normal <span class="bold">texto negrito<span> mais texto...<p>

Unidades de Medida

Pixel

Um pixel representa um ponto único na tela. Em monitores, 1 pixel geralmente corresponde ao menor elemento que pode ser controlado naquele dispositivo.

Por exemplo, se temos uma div com width: 10px, ela terá uma largura fixa de 10 pixels, independente do tamanho ou zoom da tela.

Alguns casos onde os pixels são úteis:

  • Definir a largura de bordas: como as bordas geralmente são finas (1 a 3 pixels), os pixels permitem um controle preciso.
  • Tamanhos de fontes pequenas: para textos auxiliares, usar tamanho em pixels (ex: 12px) permite consistência.
  • Elementos de tamanho conhecido e fixo: em alguns casos, sabemos exatamente o tamanho que um elemento deve ter, para encapsular um conteúdo específico.

REM

REM significa "Root EM" e é uma unidade relativamente recente que veio para simplificar o uso de medidas responsivas.

O REM baseia o tamanho do elemento no tamanho da fonte definida no elemento raiz (root) do documento HTML. Por exemplo:

div { font-size: 16px; /* Tamanho padrão da fonte nos navegadores */}
section { width: 3rem; // 3 * 16px = 48px}

Como a fonte padrão nos navegadores geralmente é de 16px, 1rem corresponde a 16px. Então se usarmos 3rem, teremos 3 * 16 = 48px. A vantagem do REM é que, se o usuário alterar o tamanho de fonte padrão no navegador para facilitar a leitura, todo o layout se adaptará proporcionalmente.

Então se o root font-size passar para 20px, 3rem será equivalente a 60px. Isso faz do REM uma unidade adaptável. Por isso ela é muito utilizada para definir espaçamentos, tamanhos de container, larguras de colunas etc.

Usando valores fracionados

Também podemos usar valores fracionados em REM, como 1.5rem, 2.25rem etc. Nesses casos, a fração é calculada em cima do tamanho base de 16px (ou do root font size definido).

Então 2.5rem com root de 16px = 2.5 * 16 = 40px. Isso permite ajustar os tamanhos com mais precisão.

EM

A unidade EM funciona de forma muito parecida com o REM, mas ao invés de se basear no font size raiz, o EM utiliza como referência o font size do próprio elemento. Exemplo:

section font-size: 16px;
button { padding: 1em; // 1 * 16px do font-size do section}

Novamente, se o font-size do elemento "pai" for alterado, o em se adapta automaticamente.

Isso faz do EM uma unidade útil quando queremos que o elemento filho tenha um tamanho proporcional ao elemento pai. Como em botões com padding relativo ao text size, por exemplo.

Porcentagem

A unidade porcentagem (%) em CSS é relativa ao elemento pai, similar ao EM. A diferença é que a porcentagem se baseia no width (largura) ou height (altura) do elemento pai, enquanto o EM usa o font size. Exemplo:

main width: 800px;
section width: 50%; // 50% de 800px = 400px
parent height: 100px;
el { height: 25% // 25% de 100px = 25px }

Novamente vemos que a porcentagem cria uma proporcionalidade entre elementos pai e filho: se redimensionamos o elemento pai, os elementos filhos mudarão proporcionalmente.

Viewport: VW e VH

As unidades VW e VH são relativas à viewport, que é a área visível da página no navegador.

  • VW = Viewport Width
  • VH = Viewport Height

1VW representa 1% da largura total da viewport.1VH representa 1% da altura total da viewport. Exemplo:

div { width: 50vw; // 50% da largura da viewport height: 75vh; // 75% da altura da viewport }

Essas unidades são ideais para criar elementos com largura ou altura relativa ao tamanho da janela do navegador, o que é útil especialmente para layouts responsivos. Alguns casos de uso:

  • Hero images e banners que ocupam toda a viewport em altura
  • Layouts com colunas que devem manter proporção em diferentes resoluções
  • Elementos grandes que precisam limitar seu tamanho à viewport (imagens, vídeos etc)

Resumindo

  • Pixels: controle preciso para tamanhos fixos e pequenos elementos
  • REM: tamanhos proporcionais ao root, ideal para espaçamentos e layouts responsivos
  • EM: tamanhos relativos e proporcionais ao elemento pai
  • Porcentagem: proporcional ao width/height do elemento pai
  • VW | VH: proporcional à largura e altura da viewport
